## 设计原则

- 数据库版本：MySQL 8.0+ (InnoDB)
- 数据库字符集：utf8mb4
- 表名命名规范：英文字母 + 下划线，如 `business_transaction`
- 字段命名规范：英文字母 + 下划线，如 `customer_id`

## 数据总体设计

本系统围绕"客户—账户—交易"三层核心展开，辅以会计核算、现金管理、利率配置、日终处理和结息审计五个支撑模块，共计 8 张表。

- **客户与账户**：一位客户可开立多个活期存款账户，通过 `customer_id` 关联。证件类型 + 证件号码唯一确定客户身份。
- **账户与交易**：存款、取款、转账、结息、销户等所有资金变动统一记录在业务交易流水表。每笔交易产生一借一贷两条会计分录，满足复式记账要求。柜面渠道的交易同步记录现金尾箱出入库明细。
- **转账**：行内转账生成两条流水（转出借方 + 转入贷方），共用一个 `trans_no`，通过 `related_trans_id` 互相关联。
- **利率与结息**：利率按账户类型和币种配置，支持生效/失效日期。每日凌晨 1 点生成日积数快照，季度末月 21 号汇总积数、匹配利率、计算并派发利息。每次结息记录完整审计信息，关联利息派发流水。
- **并发控制**：账户表采用乐观锁（`version` 字段）应对高并发余额更新，冲突时重试最多 3 次。幂等通过 Redis 实现，10 分钟 TTL。

## 附录：结息公式

```
利息(元) = 积数总和 × 日利率
积数总和 = SUM(daily_balance.end_balance) WHERE balance_date BETWEEN startDate AND endDate
计息天数 = endDate - startDate + 1
startDate = last_settlement_date（含）或 open_date
endDate = 昨天（结息日当天不参与计息）
结果四舍五入保留 2 位小数（HALF_UP）
```
